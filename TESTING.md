# 测试与验证说明

本文档用于记录“食刻点餐”项目的构建验证、线上功能核查和课程提交前的人工测试流程。

## 1. 本地构建验证

首次运行项目前安装依赖：

```bash
pnpm install
```

执行 H5 构建：

```bash
pnpm run build:h5
```

构建成功后，产物会输出到 `dist/build/h5`，可用于部署到静态托管服务。

## 2. CI 自动构建

项目已配置 GitHub Actions 工作流：

```text
.github/workflows/ci.yml
```

当代码推送到 `master` 分支，或向 `master` 发起 Pull Request 时，CI 会自动执行：

1. 拉取代码
2. 安装 Node.js 22 和 pnpm
3. 使用 `pnpm install --frozen-lockfile` 安装依赖
4. 执行 `pnpm run build:h5` 验证项目是否可以正常构建

该流程用于证明项目具备基础工程化验证能力，可作为课程加分项中的 CI 构建检查材料。

## 3. 线上功能核查

线上 Demo 地址：

[https://env-00jy6k9lf709-static.normal.cloudstatic.cn/index.html#/](https://env-00jy6k9lf709-static.normal.cloudstatic.cn/index.html#/)

建议按以下流程进行人工验收：

- 首页：检查门店信息、推荐商品、分类入口、搜索入口是否正常展示。
- 商品：检查分类筛选、商品详情、规格选择、加入购物车流程。
- 购物车：检查商品数量增减、删除、清空和结算入口。
- 订单：检查创建订单、订单列表、订单详情、取消订单、完成订单、删除订单。
- 支付：检查模拟支付方式、支付倒计时、支付结果页和订单状态变化。
- 评价：检查订单评价、商品评价列表、我的评价和删除评价。
- 地址：检查新增、编辑、删除、设置默认地址。
- 用户：检查注册、登录、切换账号、头像和昵称维护。
- 优惠与会员：检查优惠券、会员权益、会员订单记录。
- 钱包：检查开通钱包、余额、账单、模拟充值、提现、钱包支付和支付密码。
- 设置：检查通知设置、账号安全、主题、语言、适老模式、字号调整和法律信息页面。

## 4. 验收标准

- 页面可以正常打开，不出现空白页或明显资源加载失败。
- 主要路由之间可以正常跳转，返回逻辑符合移动端使用习惯。
- 购物车、订单、支付、评价、地址、钱包等核心流程可以形成闭环。
- 云函数调用有明确返回结果，失败时有提示，不阻塞页面继续使用。
- 模拟支付、模拟充值和模拟提现不涉及真实资金结算，仅作为课程项目业务流程展示。
