# 个人实训总结报告

本次《AI 辅助编程与工程化实训》中，我独立完成了“食刻点餐”uni-app 项目的设计、开发、后端接入、部署和文档整理。项目以移动端点餐场景为核心，围绕用户从浏览商品、选择规格、加入购物车、提交订单、模拟支付、查看订单到评价反馈的完整流程展开，同时补充了地址管理、商品收藏、优惠券、会员服务、食刻钱包、个人资料、通知设置、多语言、适老模式和字号调整等功能。通过这次实训，我对一个前后端结合项目从原型功能到工程化交付的全过程有了更清晰的认识。

本项目中遇到的最大困难，是如何把早期依赖前端本地缓存和模拟数据的功能，逐步改造成可以通过 uniCloud 云函数和云数据库保存真实业务数据的版本。最开始，购物车、订单、收藏、地址、会员和钱包等数据更多依赖本地 `uni.setStorageSync` 保存，这种方式做页面演示比较快，但无法体现真正的后端能力，也不利于多端数据同步。为了解决这个问题，我先梳理了项目中的核心业务模块，再按照“用户、商品、购物车、订单、地址、收藏、优惠券、会员、钱包”等维度设计数据库集合和 schema，随后分别实现 `user-auth`、`product-service`、`order-service`、`address-service`、`favorite-service`、`benefit-service`、`wallet-service` 等云函数。前端则通过 `src/utils` 下的业务封装文件统一调用云函数，尽量减少页面层的重复逻辑。这个过程让我意识到，后端化不只是把数据从本地搬到云端，更重要的是重新梳理数据结构、状态流转、异常处理和前后端职责边界。

另一个比较明显的困难是订单、支付、评价、钱包和会员之间存在较多状态联动。例如订单需要处理待支付、制作中、已完成、已取消等状态；支付流程既包含微信/支付宝模拟支付，也包含食刻钱包支付；钱包又涉及余额、账单、充值、提现、支付密码、优惠减免等逻辑。为避免功能之间互相影响，我在开发时尽量把共用逻辑拆到工具文件中，例如订单相关逻辑集中在 `order-backend.js` 和 `order-service`，钱包相关逻辑集中在 `wallet.js` 和 `wallet-service`。在评价功能中，我还处理了“已完成订单可评价”“商品详情页展示评价”“删除订单不影响已评价记录”等细节。通过反复调试这些流程，我对业务状态设计的重要性有了更深体会。

在这次实训中，AI 工具对我的帮助主要体现在三个方面。第一，AI 帮助我拆解复杂需求。面对功能比较多的点餐项目时，如果只凭直觉开发，很容易遗漏边界情况。通过向 AI 描述目标功能，我可以更快得到模块拆分、数据结构和实现步骤方面的建议。第二，AI 帮助我发现代码和文档中的问题。例如在整理 README、API 文档和 Code Review 报告时，AI 提醒我需要说明部署地址、运行步骤、uniCloud 配置、云函数 action、数据库集合以及安全注意事项。第三，AI 帮助我进行代码审查和复盘。通过 AI Code Review，我发现项目虽然功能完整，但仍存在一些工程化改进空间，例如 README 中不应引导公开填写云空间密钥、后端接口如果用于真实生产环境还需要更严格的 token/session 鉴权、项目目前缺少自动化测试等。这些建议让我不只是关注“能不能跑”，也开始关注“是否安全、是否可维护、是否方便别人复现”。

我在项目中的具体贡献包括：完成前端页面结构和主要交互设计，搭建首页、菜单、商品详情、购物车、结算、支付、订单、评价、个人中心、设置、钱包、会员等页面；完成 uniCloud 云函数的设计与接入，使登录注册、商品数据、购物车、订单、地址、收藏、优惠券、会员、评价和钱包等模块具备后端存储能力；设计并维护数据库 schema；整理项目 README、API 文档和 AI Code Review 报告；完成 H5 构建和线上 Demo 部署。除此之外，我也持续通过 Git 记录开发过程，保证项目不是最后一次性提交，而是能体现逐步迭代的过程。

通过这次实训，我最大的收获是对“工程化”有了更实际的理解。以前做项目时，我更关注页面是否美观、功能按钮是否能点通；这次则更加关注项目结构、接口文档、部署地址、Git 记录、数据模型、异常处理和可复现性。尤其是居家自学形式下，所有成果都需要通过代码、文档、提交记录和线上 Demo 来证明，因此过程留痕和文档完整性变得非常重要。AI 能提高开发效率，但它不能替代我对业务逻辑的判断，也不能替代我对代码最终质量负责。只有把 AI 的建议和自己的理解结合起来，反复验证、修改和整理，项目才能真正成为一个可以交付和说明清楚的作品。

总体来说，本次实训让我完成了一个较完整的 uni-app 点餐项目，也让我体验了从前端页面开发到云函数后端化、从功能实现到文档交付的完整流程。后续如果继续优化，我会优先补充自动化测试、完善 token 鉴权机制、优化 README 中的安全说明，并继续整理 Prompt 日志和演示材料，让项目在功能完整的基础上更加规范、可靠和可复现。
